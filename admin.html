Dados do repositório
🔒 Salvo só no seu navegador, nunca publicado no site.
Como criar o Token — passo a passo
  1. Abra github.com → clique na sua foto → Settings
  2. Role até o final da página → Developer settings
  3. Personal access tokens → Tokens (classic)
  4. Generate new token (classic)
  5. Note: escola-admin · Expiration: No expiration
  6. Marque apenas a caixa repo (primeira opção)
  7. Clique Generate token → copie o token gerado
  8. Cole no campo acima e clique Salvar ✓
Como funciona

Ao clicar Publicar, o Admin salva um arquivo data.json no seu repositório GitHub. O site público (index.html) lê esse arquivo ao carregar e exibe tudo que você configurou aqui — cursos, categorias, cores, textos e imagens.

🔗 Integração Kiwify

Conecte sua conta Kiwify para importar seus produtos automaticamente. Configure uma vez e sincronize quando quiser.

Necessário para contornar o bloqueio CORS. Siga o guia abaixo para criar o proxy gratuito.

Passo 1 — Credenciais Kiwify:
1. Acesse dashboard.kiwify.com.br
2. Menu lateral → Apps → API → Criar API Key
3. Copie o Client ID, Client Secret e Account ID

Passo 2 — Proxy gratuito (Cloudflare Worker):
1. Acesse workers.cloudflare.com e crie uma conta gratuita
2. Clique em Create Worker
3. Apague o código de exemplo e cole o conteúdo do arquivo kiwify-proxy-worker.js
4. Clique em Deploy
5. Copie a URL gerada (ex: https://xxx.workers.dev) e cole acima
6. Clique em Salvar e depois Testar conexão ✓

☁️ Backup automático no Google Drive

A cada publicação, salva automaticamente um backup no seu Google Drive. Configure uma vez.

Veja o guia abaixo para criar em 5 minutos.
Como criar o Client ID (5 minutos): 1. Acesse console.cloud.google.com
2. Crie um projeto novo (ex: kerigma-backup)
3. Menu → APIs e Serviços → Ativar APIs → ative Google Drive API
4. Menu → Credenciais → Criar credencial → ID do cliente OAuth
5. Tipo: Aplicativo da Web
6. Em "Origens JavaScript autorizadas" adicione: https://kerigmaacademy.com.br
7. Copie o ID do cliente e cole acima
